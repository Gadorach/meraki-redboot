# First hardware acceptance plan

The generated image changes the first-stage boot path. Use an external SPI
programmer and a verified full-flash backup for initial tests.

## Before writing

1. Read the complete NOR at least twice and compare SHA-256 hashes.
2. Preserve the original first `0x40000` bytes separately.
3. Build all three profiles and archive ELF, map, symbols, manifest, compiler
   version, and final SHA-256.
4. Run `make validate` and inspect descriptor offsets with `make inspect`.
5. Verify the kernel payload with `tools/mkvcoreiii_payload.py verify`.
6. Confirm payload load address and entry point match the intended kernel type.
7. Re-read the programmed first `0x40000` bytes before applying power.

## Staged tests

1. Start with `VARIANT=strict` and a payload generated by the included packer.
   This tests the source build, wrapper, CRC format, and stock-style rejection.
2. Capture complete serial output from power-on at 115200 8N1.
3. Test `VARIANT=development` with a deliberately incorrect CRC and verify that
   the warning appears before Linux starts.
4. Test the legacy-size warning only with a deliberately lowered
   `LEGACY_PAYLOAD_LIMIT`; do not enlarge the actual payload beyond the hard slot
   boundary.
5. Test `VARIANT=permissive` with the same known-good payload and verify normal
   copy and entry behavior.
6. Confirm fallback behavior with an invalid magic, zero length, unaligned
   length, and size above the hard limit before considering production use.

## Expected early serial evidence

A successful loader reaches initialization messages including the
`LinuxLoader built ...` banner and PLL, SPI, memory-controller, DRAM training,
IRQ, cache, and board stages. Failure before that banner suggests wrapper/vector
execution. Failure after initialization but before Linux points toward payload
header, addresses, size, CRC policy, or entry point.

Warning builds should emit one of:

```text
WARNING: payload CRC mismatch; continuing because CRC policy is warn
WARNING: payload exceeds legacy size threshold; continuing within hard slot limit
```

## Current limitation

The wrapper and policies have been structurally linked and validated, but the
new image has not yet been executed on physical MS220/MS42 hardware in this
environment. Hardware acceptance remains required before production use.
